JIRAPublisher

The JIRAPublisher component creates a new JIRA issue whenever a build fails. The new issue will be assigned to the person responsible for breaking the build. (If DamageControl can't figure out who broke the build, it will pick the first person in the set of committers as asignee).

Requirements

JIRA 1.4 or later installed somewhere, with Jelly support enabled. See the JIRA documentation for details.

Configuration

JIRA Configuration

You should create a separate JIRA user for DamageControl. This will be the "reporter" user for new JIRA issues when a build fails.
It is recommended that the JIRA user is called "damagecontrol" and that the email be set to that of one of the DamageControl administrators.

The damagecontrol JIRA user must also be granted privileges to create new issues for all JIRA projects that wish to let the JIRAPublisher post new issues for a particular project.

DamageControl Server Configuration

Make sure you add the following line to the DamageControl server startup script:

# The root URL of the JIRA installation
jira_url = "http://jira.codehaus.org/"
JIRAPublisher.new(@hub, ShortTextTemplate.new, jira_host).start

Individual project configuration

The project specific JIRA information should go into the DamageControl configuration file (someproject-damagecontrol.conf). Example:

jira_project_key: DC
